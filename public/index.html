Why DarkMatter exists

Three problems nobody else has solved.

Problem 1
Agent outputs are unverifiable by design

Every record was written by a party with an interest in the outcome. When a regulator asks "prove what your agent did," there is no independent source of truth.

DarkMatter: Payload hashes are computed client-side. Agents sign the full commit envelope. Tampering is detectable without trusting DarkMatter.

Problem 2
Multi-agent pipelines have no chain of custody

When Agent A passes work to Agent B to Agent C, nothing records the handoffs. If the output is wrong, you cannot trace where the error was introduced.

DarkMatter: Every record links to its parent. Walk from any output back to its root input — across any model, framework, or language.

Problem 3
You can't investigate a decision without re-running it

To test what would have happened differently at step 3, you re-run the entire pipeline from scratch — with no guarantee of identical conditions.

DarkMatter: Branch from any checkpoint. Run an alternative. Compare the two records step by step.

How it works

How a decision becomes a verifiable record

Build agents with any framework. DarkMatter records the execution.

01
The agent produces an output
The payload is hashed client-side and signed by the agent with its own Ed25519 key. Nothing leaves the agent unsigned. The signing happens before transmission — DarkMatter never sees the pre-hash payload.
02
The record is appended
The signed step is added to an append-only chain with a Merkle inclusion proof. Every step links cryptographically to the one before it. Any modification breaks every step that follows.
03
The record can be verified
Anyone can verify the full execution — offline, without trusting DarkMatter. Export a verifiable JSON proof bundle. Share the standalone verifier. Your auditor needs no account and no access to your system.
