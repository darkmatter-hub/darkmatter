Four primitives

Everything your pipeline needs to be accountable.

No new orchestration layer. Wrap one function call and the rest follows.

dm.commit()
Commit
Record every execution step as a cryptographically chained context. SHA-256 payload hash linked to its parent. Tamper-detection built in.
dm.replay()
Replay
Walk the full decision chain from root to tip. See exactly what each agent received and produced, in sequence, with verified integrity at every step.
dm.fork()
Fork
Branch from any checkpoint. Run the same pipeline with a different model, prompt, or agent. The original chain is always intact.
dm.verify()
Verify
Export a self-contained proof bundle. Anyone can verify integrity using the included verifier script — no DarkMatter account, no internet connection.

How it works

Three steps. No new infrastructure.

01
Wrap your model call with dm.commit()
One function wraps any model call. DarkMatter creates a chained context record with a SHA-256 payload hash and links it to the parent step.
ctx = dm.commit(to_agent_id, payload={"output": result})
# → ctx["id"] = "ctx_1775..." — every step gets a permanent, replayable ID
02
Replay or fork from any checkpoint
Walk the full chain, branch from any step, or diff two branches. The original is always intact. No reruns unless you choose to.
replay = dm.replay(ctx_id)   fork = dm.fork(ctx_id)   diff = dm.diff(a, b)
03
Export a verifiable proof bundle
Export a self-contained artifact with chain records, hashes, and a standalone verifier. Anyone can check it — no DarkMatter account needed. Verification works offline.
bundle = dm.export(ctx_id)  # → proof_bundle.zip + verifier.py

Verification

Verification does not require trusting DarkMatter.

Export bundles are self-contained.
Exported bundles contain all chain records, payload hashes, and a standalone verifier script. The verifier checks integrity using Python stdlib only — no network calls, no account, no internet connection. Works offline. Will work in five years regardless of whether DarkMatter exists.

Self-hosting is also supported under MIT license. Note: when you control the infrastructure, the record is no longer stored outside your system — the third-party independence guarantee only applies to the cloud product. Both are legitimate; they have different trust models.
python verifier.py proof_bundle.zip — pass/fail per step · no dependencies beyond Python stdlib
Context Passport — open standard.
DarkMatter implements the Context Passport schema — an open, CC0-licensed standard for AI agent context handoffs. The schema is actor-agnostic: human-to-agent and agent-to-agent handoffs use the same format. You can verify any Context Passport chain with any compatible verifier.

The format is documented. Each commit stores a payload_hash and integrity_hash chained to its parent via SHA-256. Build your own verifier if you prefer.
