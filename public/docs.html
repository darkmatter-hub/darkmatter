DarkMatter Docs

Execution history for AI agent pipelines. Every commit, every handoff, every fork — immutable and verifiable.

Base URL: https://darkmatterhub.ai  ·  Get an API key →

Quickstart

Three commands to get from zero to a verified execution chain.

1
Get an API key
Sign up at darkmatterhub.ai/signup, create an agent in the dashboard, and copy the API key.
2
Commit your first context
Call POST /api/commit after any agent step. Pass parentId to chain commits together.
3
Replay the chain
Call GET /api/replay/:ctxId to walk the full decision path root to tip with integrity verified at every step.
# 1. Commit context from Agent A to Agent B
curl -X POST https://darkmatterhub.ai/api/commit \
  -H "Authorization: Bearer YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "toAgentId": "dm_agent_b_id",
    "payload": {
      "input":  "Analyze Q1 earnings",
      "output": "APAC up 34%, margin compressed"
    },
    "agent": { "role": "researcher", "model": "claude-opus-4-6" }
  }'
# Returns: { "id": "ctx_...", "integrity": { "payload_hash": "sha256:..." } }

# 2. Agent B pulls verified context
curl https://darkmatterhub.ai/api/pull \
  -H "Authorization: Bearer AGENT_B_KEY"

# 3. Replay full chain (pass parent ctx id to chain)
curl https://darkmatterhub.ai/api/replay/CTX_ID \
  -H "Authorization: Bearer YOUR_KEY"

Core concepts

Context commit

The fundamental unit. Every agent step that you want to track becomes a context commit — an immutable node with a payload, agent attribution, and integrity hash. Once committed, it cannot be modified.

Parent linkage

Pass parentId in each commit to link it to the previous step. This builds the lineage graph. Without parentId, a commit is a root node.

Integrity chain

Each commit stores payload_hash = sha256(payload) and integrity_hash = sha256(payload_hash + parent_integrity_hash). Tamper with any node and every downstream hash breaks. Use GET /api/verify to check.

Fork

Branch from any checkpoint without touching the original chain. Like git branch. The fork carries fork_of, fork_point, and lineage_root fields so you always know where it came from.

Authentication

All API endpoints require a Bearer token:

Authorization: Bearer dm_sk_your_api_key_here

Get your API key from the dashboard. Each agent has its own key. Rate limit: 120 requests per minute per key.

Integration: LangGraph

Call DarkMatter after each node in your LangGraph graph. Pass the returned ctx_id as parentId in the next commit to build the chain.

# Python — add to any LangGraph node
import requests

DM_URL = "https://darkmatterhub.ai"

def commit_to_darkmatter(api_key, to_agent_id, input_data, output_data, parent_id=None):
    body = {
        "toAgentId": to_agent_id,
        "payload": { "input": input_data, "output": output_data },
    }
    if parent_id:
        body["parentId"] = parent_id
    res = requests.post(
        f"{DM_URL}/api/commit",
        headers={"Authorization": f"Bearer {api_key}"},
        json=body
    )
    return res.json()

# In your LangGraph node:
def researcher_node(state):
    output = llm.invoke(state["input"])
    ctx = commit_to_darkmatter(
        api_key=AGENT_A_KEY,
        to_agent_id=AGENT_B_ID,
        input_data=state["input"],
        output_data=output.content,
        parent_id=state.get("dm_parent_id")
    )
    return {"output": output.content, "dm_parent_id": ctx["id"]}
Note: LangGraph has its own internal checkpointing. DarkMatter adds cross-framework lineage and cryptographic proof on top of that — they work alongside each other, not instead of each other.

Integration: Raw API calls

Works with any model or framework. Wrap your API calls and commit before and after.

import anthropic, requests

client = anthropic.Anthropic()
DM_KEY = "dm_sk_your_key"
NEXT_AGENT_ID = "dm_next_agent_id"

def run_and_commit(prompt, parent_id=None):
    # Run the model
    response = client.messages.create(
        model="claude-opus-4-6", max_tokens=1024,
        messages=[{"role": "user", "content": prompt}]
    )
    output = response.content[0].text

    # Commit to DarkMatter
    body = {
        "toAgentId": NEXT_AGENT_ID,
        "payload": {"input": prompt, "output": output},
        "agent": {"provider": "anthropic", "model": "claude-opus-4-6"},
    }
    if parent_id:
        body["parentId"] = parent_id
    ctx = requests.post(
        "https://darkmatterhub.ai/api/commit",
        headers={"Authorization": f"Bearer {DM_KEY}"},
        json=body
    ).json()

    return output, ctx["id"]  # pass ctx id to next step as parent

Integration: CrewAI

Add a DarkMatter commit in your task callback or after each agent finishes.

from crewai import Agent, Task, Crew
import requests

def after_task(task_output, agent_key, to_agent_id, parent_id=None):
    body = {
        "toAgentId": to_agent_id,
        "payload": {
            "input":  str(task_output.task.description),
            "output": str(task_output.raw)
        },
    }
    if parent_id: body["parentId"] = parent_id
    return requests.post(
        "https://darkmatterhub.ai/api/commit",
        headers={"Authorization": f"Bearer {agent_key}"},
        json=body
    ).json()["id"]

Integration: Local models (Ollama, llama.cpp, vLLM)

import requests as req

def run_local_and_commit(prompt, model, dm_key, to_agent_id, parent_id=None):
    # Run local model via Ollama
    response = req.post("http://localhost:11434/api/generate",
        json={"model": model, "prompt": prompt, "stream": False}
    ).json()["response"]

    # Commit to DarkMatter
    body = {
        "toAgentId": to_agent_id,
        "payload": {
            "input":  prompt,
            "output": response,
            "memory": {"model": model, "provider": "local"}
        },
        "agent": {"provider": "local", "model": model},
    }
    if parent_id: body["parentId"] = parent_id
    ctx = req.post("https://darkmatterhub.ai/api/commit",
        headers={"Authorization": f"Bearer {dm_key}"}, json=body
    ).json()
    return response, ctx["id"]

POST /api/commit

Commit agent context. Returns a canonical v2 context object.

{
  "toAgentId":  "dm_...",           // required — recipient agent ID
  "payload": {                    // structured state
    "input":     "...",
    "output":    "...",
    "memory":    {},
    "variables": {}
  },
  "parentId":   "ctx_...",        // optional — chains to parent
  "traceId":    "trc_...",        // optional — groups commits into a run
  "eventType":  "commit",         // commit|fork|checkpoint|revert|...
  "agent": {
    "role":     "researcher",
    "provider": "anthropic",
    "model":    "claude-opus-4-6"
  }
}

GET /api/pull

Pull all verified contexts addressed to the authenticated agent. Returns newest first.

GET /api/replay/:ctxId

Walk the parent chain from ctxId back to root, reverse to chronological order, return each step with full payload and integrity verification.

Add ?mode=summary for metadata only (no payloads, faster).

POST /api/fork/:ctxId

Branch from any checkpoint. Returns a new context ID with fork_of, fork_point, and lineage_root. Continue the branch by committing with parentId set to the new ID.

GET /api/verify/:ctxId

Returns a standalone trust object: chain_intact, length, root_hash, tip_hash, broken_at, fork_points, verified_at.

GET /api/export/:ctxId

Downloads a portable JSON proof artifact. Contains full chain, integrity block, chain_hash (stable across exports of the same chain), and export_hash (unique per export).

Enterprise: BYOK encryption

Available on Enterprise plans. Your payload is encrypted with AES-256-GCM using a key you provide. DarkMatter stores only ciphertext.

# Encrypted commit
curl -X POST https://darkmatterhub.ai/enterprise/commit \
  -H "Authorization: Bearer YOUR_KEY" \
  -d '{
    "toAgentId": "dm_...",
    "payload": {"input": "...", "output": "..."},
    "byokKey": "your-64-char-hex-aes256-key"
  }'
Security note: Your BYOK key is never stored by DarkMatter. Pass it in each request. Store it in your secrets manager (AWS KMS, HashiCorp Vault, etc.).

Enterprise: W3C DID agent identity

Register a W3C Decentralized Identifier for any agent. Commits from that agent will carry a verifiable DID, independently resolvable without trusting DarkMatter.

# Register DID for an agent
curl -X POST https://darkmatterhub.ai/enterprise/did/register \
  -H "Authorization: Bearer DASHBOARD_TOKEN" \
  -d '{
    "agentId":   "dm_...",
    "didId":     "did:web:yourcompany.com:agents:researcher",
    "publicKey": "z6Mk..."
  }'

# Resolve DID document (public, no auth)
curl https://darkmatterhub.ai/enterprise/did/AGENT_ID
